# Security Policy

## Reporting a Vulnerability

At VilChain, we take security seriously. If you believe you have found a security vulnerability in our project, we encourage you to let us know right away. We appreciate the efforts of security researchers in keeping our project and users safe.

To report a security vulnerability, please follow these steps:

1. **Privately Disclose:** Please avoid public disclosure of the vulnerability until we have had a chance to address it.

2. **Submit a Report:**
   - Email: security@vilchain.com
   - Subject: [Security] Vulnerability in VilChain

3. **Provide Details:**
   - Clearly describe the vulnerability.
   - Include steps to reproduce the vulnerability.
   - Attach any supporting material like proof-of-concept code, screenshots, or logs.

4. **Responsible Disclosure:**
   - Allow us a reasonable amount of time to address the issue before disclosing it publicly.
   - We will keep you informed of our progress and coordinate the release of any security advisories.

## Bug Bounty

We appreciate the efforts of security researchers and offer a bug bounty program for responsibly disclosed security vulnerabilities. The program details, eligibility, and rewards can be found on our [Bug Bounty Program](BUG_BOUNTY.md) page.

## Scope

The security policy and bug bounty program are limited to vulnerabilities discovered in the VilChain project. Third-party dependencies or services should be reported to the respective maintainers or vendors.

Thank you for helping us make VilChain more secure!
